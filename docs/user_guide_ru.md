# Руководство пользователя VPS Capture

Это руководство описывает полный цикл работы в приложении: от первой настройки до записи выступлений, включая механизм **«Отложить»** и возврат участника обратно в работу.

## 1) Для чего приложение

VPS Capture — это десктоп-приложение для соревнований, где нужно быстро отмечать выступления и сохранять клипы по участникам.

Как это работает в целом:
- приложение держит непрерывный «кольцевой» буфер из коротких `.ts`-сегментов;
- оператор нажимает **START** в момент начала выступления;
- оператор нажимает **STOP** в момент окончания;
- приложение собирает клип из сегментов между START и STOP и сохраняет его в папку вывода.

## 2) Что подготовить до старта

Перед работой проверьте:
- установлен/доступен `ffmpeg` (и желательно `ffplay` для превью);
- подключено видеоустройство (камера, карта захвата, DeckLink и т.д.);
- (по необходимости) подключено аудио;
- подготовлен файл расписания в текстовом формате.

## 3) Экран «Настройки» (Setup)

### 3.1 Язык интерфейса
В левом верхнем блоке можно выбрать язык интерфейса: `EN / RU / TR`.

### 3.2 ffmpeg / ffplay
Есть три сценария:
1. **Установить автоматически** — приложение пытается скачать/подготовить бинарники.
2. **Выбрать путь к ffmpeg...** — вручную указать исполняемый файл.
3. **Выбрать путь к ffplay...** — вручную указать путь для окна превью.

### 3.3 Превью и тестовая запись
- **Запустить превью** — проверка «живого» сигнала.
- **Запустить тестовую запись** — короткий тест записи до начала соревнований.

### 3.4 Папка вывода
Кнопкой **Выбрать папку вывода** задаётся корневая папка, куда будут складываться финальные клипы.

### 3.5 Источник захвата и устройства
1. Выберите тип источника (зависит от ОС и оборудования).
2. Нажмите **Сканировать устройства**.
3. Выберите видеоустройство и (если нужно) аудиоустройство.

### 3.6 Параметры записи
- **Кодек** (например, `h264_videotoolbox`, `libx264` и др.).
- **Буфер, мин** — глубина кольцевого буфера.
- **Преролл, сек** — дополнительный захват «до старта» при экспорте клипа.

### 3.7 Переход к рабочему экрану
Кнопка **Перейти к рабочему экрану** становится доступной после базовой настройки и выбора устройств.

## 4) Расписание: форматы и загрузка

Расписание можно загрузить из файла или вставить вручную на рабочем экране.

Базовый формат строки участника:
- `ФИО;ГОРОД;СНАРЯД`  
(поле «СНАРЯД» необязательное)

Примеры:
- `Иванов Иван;Москва;Кольца`
- `Петров Петр;Казань`

### Блочная структура потоков/видов
Если строка начинается с `/*`, она воспринимается как заголовок блока.  
Из заголовка может считываться число «видов/ротаций». Например:
- `/*2 Поток 1:` — значит для списка участников блока будет создано 2 типа/вида.

Практический совет: используйте единый шаблон файла расписания по площадке и проверьте его на 2–3 участниках до старта трансляции.

### Дополнительные примеры файлов импорта

Ниже несколько готовых вариантов, которые можно сохранить как `.txt` или `.csv` и загрузить в приложение.

Эти же примеры добавлены в репозиторий отдельными файлами:
- `docs/import_examples/ru_example_01_simple.txt`
- `docs/import_examples/ru_example_02_one_thread_two_types.txt`
- `docs/import_examples/ru_example_03_two_threads_mixed_types.txt`
- `docs/import_examples/ru_example_04_with_empty_lines.txt`

**Пример 1 — простой список без блоков**
```
Иванова Анна;Москва;Бревно
Петрова Мария;Казань;Брусья
Сидорова Елена;Пермь
```

**Пример 2 — один поток, 2 вида (дублирование участников по видам)**
```
/*2 Поток 1: юниорки
Кузнецова Дарья;Тюмень;Опорный прыжок
Орлова София;Екатеринбург;Вольные
```

**Пример 3 — два потока с разным количеством видов**
```
/*1 Поток 1: мальчики
Иванов Максим;Омск;Кольца
Сергеев Артём;Новосибирск;Конь

/*3 Поток 2: девочки
Морозова Алиса;Сочи;Бревно
Егорова Полина;Краснодар;Брусья
```

**Пример 4 — файл с пустыми строками и смешанным заполнением «снаряда»**
```
/*2 Поток А
Алексеева Вера;Самара;Брусья

Николаева Инна;Уфа

/*1 Поток Б
Фёдорова Ольга;Челябинск;Вольные
```

Советы по импорту:
- используйте разделитель `;` (точка с запятой);
- избегайте лишних символов в начале строки участника;
- если есть блоки `/*...`, сначала укажите заголовок блока, затем участников этого блока.

## 5) Экран «Работа» (Work)

На рабочем экране есть:
- список участников со статусами;
- кнопки START/STOP/POSTPONE на каждой строке;
- фильтры по потоку и виду;
- лог событий;
- опциональные GIF-титры (имя, город и выбранный шаблон).

### Статусы участника
- **В ОЧЕРЕДИ** — ещё не записывался.
- **АКТИВНО** — сейчас идёт запись (после START).
- **ГОТОВО** — клип сформирован (после STOP).
- **ОТЛОЖЕНО** — участник временно отложен.

### Горячие клавиши
- `↑ / ↓` — перемещение по списку.
- `Space` — START/STOP текущего выбранного участника.
- `Ctrl + Left Alt` — POSTPONE для текущего выбранного участника.

---

## 6) Основной сценарий записи (без отложений)

1. Выберите участника в списке.
2. Нажмите **START** (или `Space`).
3. После окончания выступления нажмите **STOP** (или снова `Space`).
4. Участник получает статус **ГОТОВО**, клип сохраняется в папку вывода.

Если экспорт не удался, запись участника будет сброшена обратно в «В ОЧЕРЕДИ».

---

## 7) Как работает «Отложить» и как вернуть участника обратно

Это ключевой блок для вашей задачи.

### 7.1 Что делает POSTPONE
При нажатии **ОТЛОЖИТЬ**:
1. Выбранный участник получает статус **ОТЛОЖЕНО**.
2. Участник перемещается в начало внутреннего списка расписания.
3. В интерфейсе отложенные показываются отдельной группой перед активными/ожидающими.

### 7.2 Если отложили активного участника
Если участник был в статусе **АКТИВНО** в момент POSTPONE:
- текущий буфер записи останавливается;
- активная отметка записи снимается;
- участник переводится в **ОТЛОЖЕНО**.

Важно: это не равно STOP-экспорту. То есть автоматического сохранения финального клипа как «ГОТОВО» при POSTPONE нет.

### 7.3 Ограничение при активной записи другого участника
Если уже есть участник в **АКТИВНО**, нельзя отложить **другого** участника, пока активная запись не завершена/снята.

### 7.4 Как вернуть отложенного участника обратно в работу
У отложенного нет отдельной кнопки «вернуть в очередь». Возврат выполняется рабочим циклом:
1. выберите отложенного участника в списке;
2. нажмите **START**;
3. после выступления нажмите **STOP**.

После этого он перейдёт в **ГОТОВО** (стандартно, как обычный участник).

### 7.5 Кнопка «Вернуть» (Undo) — для кого она
Кнопка **Вернуть** доступна у статуса **ГОТОВО** и переводит участника обратно в **В ОЧЕРЕДИ** (например, если нужно перезаписать).

## 8) Папки и файлы: что сохраняется

Приложение работает с двумя уровнями данных:

1. **Временные сегменты буфера** (`.ts`) — служебные файлы кольцевой записи.
2. **Финальные клипы** — экспортируются в папку вывода по структуре расписания/потоков.

Также состояние расписания сериализуется в `schedule.json` рядом с исполняемым файлом, чтобы при перезапуске можно было восстановить статусы.

---

## 9) Рекомендации для оператора на соревнованиях

- Перед стартом смены сделайте 1–2 тестовых клипа.
- Держите фокус на окне приложения (для горячих клавиш).
- Если участник внезапно не готов — используйте **ОТЛОЖЕНО**, это штатный сценарий.
- Если нужно перезаписать уже сохранённого — используйте **Вернуть** у **ГОТОВО** и запишите заново.
- Периодически проверяйте лог внизу экрана: там причины ошибок ffmpeg и путь сохранённых файлов.

---

## 10) Быстрый чек-лист перед реальной работой

1. ffmpeg/ffplay указаны корректно.
2. Источник захвата и устройства выбраны.
3. Папка вывода доступна для записи.
4. Расписание загружено, список отображается.
5. Превью и тестовая запись проходят успешно.
6. Оператор знает три горячие клавиши: `↑/↓`, `Space`, `Ctrl+Left Alt`.

Готово — можно вести соревнование в рабочем режиме.
